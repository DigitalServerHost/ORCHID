# 並列複数メモリ最小実証 PoC v0.1

**発案者:** 大平鉄兵 / Teppei Oohira  
**目的:** 「未分化な単一メモリ運用」と「役割分離された並列複数メモリ運用」の処理待ち差を、同一論理処理・同一出力の下で決定論的に可視化する。

## 重要な事実境界

既存PCの物理DRAMは、既に複数のチャネル／バンクを内包しています。したがって、本PoCは「現在の物理DRAMが文字通り単一の直列メモリである」と主張するものではありません。

本PoCが比較するのは、設計モデルとしての以下二つです。

| モデル | 内容 |
|---|---|
| `serial_single_memory` | 全ての読み書きが一つの未分化なメモリサービスへ直列投入される |
| `parallel_two_memory_role_split` | データ役割を分離し、独立に処理可能な読み取りを別メモリサービスへ並列投入する |

つまり、証明対象は **複数バンクが存在すること** ではなく、**処理上の役割を明示してメモリ運用を分離した場合、直列化されていた待ちがどれだけ消えるか** です。

## 最小ワークロード

STREAM Triadに近い、メモリ支配型の単純処理を使います。

```text
A[i] = B[i] + scalar * C[i]
```

各要素に必要なメモリ操作は、

```text
READ B[i]
READ C[i]
WRITE A[i]
```

の3回です。

## 比較条件

| 条件 | 利用可能メモリサービス | 役割割当 | 意味 |
|---|---:|---|---|
| `serial_single_memory` | 1 | `B,C,A -> bank0` | 既存の直列・未分化基準 |
| `parallel_two_memory_role_split` | 2 | `B,A -> bank0`, `C -> bank1` | 保守的な並列分離。定常的に約1.5倍が狙える |
| `parallel_three_memory_role_split` | 3 | `B -> bank0`, `C -> bank1`, `A -> bank2` | 独立役割を最大分離した上限参照 |
| `parallel_two_memory_conflicted_control` | 2 | `B,C,A -> bank0` | メモリが複数あっても割当を誤れば速くならない対照群 |

## 実行

```bash
python3 src/parallel_memory_minimal_poc.py --out-dir evidence/current
```

または、

```bash
chmod +x run_demo.sh
./run_demo.sh
```

## 本PoCが通れば言えること

- 同一論理処理・同一出力で、メモリ運用の役割分離のみがスケジュール完了時間を短縮する。
- 二つの独立メモリサービスだけでも、3メモリ操作のうち読み取り一組を並列化することで、直列基準に対し約1.5倍の差が出る。
- 複数メモリを置くだけでは不十分で、役割分離と割当制御が必要である。

## 本PoCだけでは言えないこと

- 既存PC上で実性能がそのまま1.5倍になること。
- 新メモリ構造の製品実装が完成したこと。
- 機械語フォーマット改善との統合で2倍が確定したこと。

物理実証の次段では、独立したメモリ経路を露出できる基板、FPGA、詳細アーキテクチャシミュレータ、又はNUMA／複数チャネルを厳密に制御できる環境が必要です。
