# 機械語／メモリ運用性能PoC 証跡パッケージ v0.1

**発案者:** 大平鉄兵 / Teppei Oohira  
**目的:** Kevin が再現・検討できる最小実証ZIP。

## このZIPが示す範囲

同一x86-64上・同一整数行列積の出力一致を維持したまま、実行順とメモリアクセス構造を、

- `I-J-K`：局所性に不利
- `I-K-J`：局所性に有利

へ変えた場合に、大きな実行時間差が生じ得ることを再現する。

既存の公平測定ログでは、速度差は次の通り。

- 最小：**1.653倍**
- 中央値：**1.692倍**
- 最大：**1.723倍**
- 平均：**1.691倍**

## このZIPだけでは示さない範囲

- 新しい機械語フォーマットの成立
- メモリ運用命令化の成立
- 全用途での2倍性能
- 製品化可能性の確定

これは、より大きな仮説に対する**第一の既存実行証跡**である。

## 本論との関係

主題はアセンブラ文法ではない。`locality_assembler.py` は既存実証スクリプトの系譜を示す名称にすぎず、本論は、

- 機械語フォーマット自体の再設計
- メモリの配置・移送・再利用・先行準備を実行制御対象へ引き上げること
- 両者の統合による追加性能差

にある。

## 実行条件

- Linux x86-64
- Python 3.10+
- GCC

## 実行

```bash
chmod +x run_demo.sh
./run_demo.sh
```

公平測定では、出力一致確認、flat/locality の先行順交互化、64MiBキャッシュ攪拌、8組のペア計測を行う。

次の検証では、`perf` / PMU カウンタ取得と、高度最適化コンパイラ又はライブラリとの公平比較が必須である。
